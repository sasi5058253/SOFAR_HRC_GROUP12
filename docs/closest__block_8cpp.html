Detailed Description

Node returning the *closest block to a given pose.

Author
Marco Gabriele Fedozzi (50833.nosp@m.65@s.nosp@m.tuden.nosp@m.ti.u.nosp@m.nige..nosp@m.it)

ServiceServer:
/block_to_pick (sofar_hbc_01::Block2Pick)

ServiceClient:
/tf/blocks (sofar_hbc_01::BlocksPoses)

Description:

This node returns the pose of a block to move the end effector toward. Despite the name, the position returned is not necessarily that of the closest blue block, but there is a simple priority system in place:

  1. If the arm passed to the request is the left one and there is a block near the center line of the workspace, return that block (since we want to free up the "common space" as soon as possible);
    1. If there are unobstructed blue blocks in the area served by the queries arm, return the one closest to the current arm pose;
    2. If there are only blue blocks obstructed by red ones in the area served by the queried arm, return the pose of the red one covering the closest blue block;
